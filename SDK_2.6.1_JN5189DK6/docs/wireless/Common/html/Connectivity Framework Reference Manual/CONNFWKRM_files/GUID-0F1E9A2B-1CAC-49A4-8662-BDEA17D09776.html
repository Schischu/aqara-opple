QN9090, JN5189 and K32W061 - Persistent Data Manager (PDM) component

For QN9090, JN5189 and K32W061, the Persistent Data Manager (PDM) component is used to store non- volatile data.

The PDM is used to store application data such as:
  • Bonding tables in Bluetooth LE applications.
  • Zigbee Application layer data:
    • AIB members, such as the EPID and ZDO state
    • Group Address table
    • Binding table
    • Application key-pair descriptor
    • Trust Center device table
    • Network layer data
    • NIB members, such as PAN ID and radio channel
    • Neighbor table
    • Network keys
    • Address Map tables
  • Additionally data concerning the Radio characterization in temperature are to be stored in the PDM
  • Customer application data if necessary
    Note: When a record is first created, a unique 16-bit identifier must be assigned to it record by the application - this identifier is then used to reference the record. The value used must not clash with those used by the NXP libraries - For Bluetooth LE-related products: - the Radio calibration uses 0xff00. - the range Bluetooth LE bonding data use the range 0x4011..0x4015. For Zigbee related products: -the ZPSNWK used 0xf100..0xf1ff - the ZPSAPL services use 0xf000..0xf0ff.
  • The range 0x0000..0x3fff is left for customer applications.
Note: The PDM section must be large enough to provide an efficient wear-leveling mechanism.
A full description of the PDM and its available APIs is described in the document JN-UG-3133-Core-Utilities.pdf.
